# ADR-003 — v1.0.3 Stakeholder-First & Korean-First 정책

- **Status:** Accepted
- **Owner Approval:** granted
- **Date:** 2026-06-09
- **Drivers (QS):** Communicability(공유·합의·전달), Maintainability, Consistency

## Context
Harness가 암묵적으로 독자를 Architect/Senior Engineer로 가정해 왔다. 실제 독자는 Lab Head·PM·CX·UX·QA·
Client/Server 개발자·기획자·임원 등 모든 stakeholder다. 또한 Cline이 Source Record를 영어로 작성하고,
산출물에 "비전문가/임원도 이해" 같은 낮춤 표현이 박히는 문제가 있었다.

## Decision
1. **독자 모델 재정의:** 설계 산출물의 독자는 아키텍트가 아니라 모든 stakeholder. 목적은 저장이 아니라 공유·합의·전달.
2. **한국어 우선:** stakeholder가 보는 모든 산출물(서술)은 기본 한국어, 기술 용어만 영어. **Source Record 포함.**
3. **낮춤 표현 금지:** "비전문가/임원도 이해/non-expert/executive-friendly/layperson" 금지.
4. **이해도 테스트(5문항)** 및 **Stakeholder Responsibilities(누가 무엇을)** 를 산출물 필수 요소로.
5. archdev에 **한국어-우선 게이트**와 **낮춤표현 금지 게이트** 추가(design/audit/self-hosted 공통, arch/ 대상).

## Alternatives considered
- 언어를 owner 재량으로만 두기 → Cline의 영어 산출 문제가 재발하므로 기각.
- 별도 "쉬운 버전" 산출물 분리 → 정보 이원화·유지비용 증가로 기각. 본 산출물 자체를 쉽게.

## Consequences
- 공유성↑, 합의·실행 연결↑. 게이트가 영어 서술·낮춤표현을 차단.
- 기존 영어 산출물은 한국어로 재작성 필요(일회성 비용).
